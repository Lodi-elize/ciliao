# 次聊 NFC 社交聊天软件 V1.0 软件说明书

## 1. 软件概述

次聊 NFC 社交聊天软件是一款面向移动端用户的社交聊天应用，主要解决线下场景中快速建立好友关系和开始一对一聊天的问题。软件通过 NFC 标签承载好友邀请信息，用户用支持 NFC 的手机触碰标签后即可读取邀请内容，进入好友申请流程；对方同意申请后，双方自动建立通讯录关系并可以进行一对一文字聊天。

本软件由移动端 App 和后端服务组成。移动端负责用户交互、NFC 读取、好友申请、通讯录展示和聊天界面；后端负责账号认证、用户资料、好友关系、消息存储、NFC 读取事件记录和实时消息推送。

## 2. 软件目标

1. 提供一种比手动搜索账号更直接的线下加好友方式。
2. 支持用户完成注册、登录、资料维护、好友申请、通讯录和聊天主流程。
3. 在 NFC 不可用的模拟器或测试环境中提供手动 invite fallback，方便调试和演示。
4. 通过后端数据持久化和实时推送，保持多设备聊天和好友关系状态一致。

## 3. 运行环境

### 3.1 移动端环境

- Android：支持 React Native 应用运行；完整 NFC 功能需要支持 NFC 的真机。
- iOS：需要真机和签名环境；NFC 读取功能依赖 iOS NDEF 能力。
- 移动端框架：Expo 54、React Native 0.81、React 19。
- 本地状态管理：Zustand、AsyncStorage。
- NFC 能力：`react-native-nfc-manager`。

### 3.2 后端环境

- Java 17。
- Spring Boot 3.5。
- Spring Web、Spring Security、Spring Data JPA、WebSocket。
- Flyway 数据库迁移。
- H2 文件数据库或 MySQL 数据库。

### 3.3 网络环境

- 移动端通过 HTTP/HTTPS 调用后端 API。
- 聊天消息和联系人变更通过 WebSocket 实时推送。
- 本地开发默认后端地址为 `http://localhost:4000`，真机测试时需要配置为局域网或公网服务地址。

## 4. 软件结构

| 层级 | 说明 |
| --- | --- |
| 移动端应用层 | React Native 页面、组件和交互逻辑 |
| 移动端业务层 | 账号、NFC、通讯录、聊天、个人资料等业务状态管理 |
| API 通信层 | 封装移动端到后端的 HTTP 请求和数据类型 |

| 实时通信层 | WebSocket 连接维护和实时事件分发 |

## 5. 主要功能说明

### 5.1 用户注册与登录

用户可以通过用户名和密码注册账号，也可以通过手机号和 mock 短信验证码注册账号。登录成功后，系统返回会话令牌，移动端保存令牌并用于后续接口访问。软件支持自动登录恢复、退出登录和修改密码。

### 5.2 个人资料管理

用户登录后可以查看和维护个人资料，包括昵称、头像和个性签名。资料更新后，联系人和聊天页面可以显示新的用户信息。

### 5.3 用户搜索

软件支持通过用户名、用户 ID 或手机号搜索其他用户。搜索结果用于发起好友申请或查看邀请目标用户信息。

### 5.4 好友申请

用户可以向目标用户发送好友申请。对方在待处理好友申请列表中可以选择同意或拒绝。若对方同意，系统创建双向联系人关系，双方均可以在通讯录中看到对方。

### 5.5 NFC 邀请加好友

NFC 标签可写入 NDEF URI 或文本记录，例如：

```text
nfcchat://add-friend?userId=bob
```

移动端读取 NFC 内容后，解析目标用户 ID，记录一次 NFC 读取事件，并进入好友申请流程。该功能适合线下见面、展会、社交活动等需要快速交换联系方式的场景。

### 5.6 手动邀请兜底

在模拟器、浏览器、无 NFC 设备或 NFC 权限不可用的环境中，用户可以手动输入 invite 内容完成同等测试流程，保证业务链路可验证。

### 5.7 通讯录管理

软件展示当前用户的好友列表。好友关系来自后端持久化数据，登录恢复后会重新拉取联系人信息，保证本地状态与后端一致。

### 5.8 一对一聊天

用户选择通讯录联系人后进入聊天页面，可以发送文本消息。后端保存消息记录，并按会话和联系人关系返回历史消息。消息发送后，对方在线时可通过实时通道收到推送。

### 5.9 实时消息推送

后端通过 WebSocket 维护实时连接。新消息、联系人变更等事件会推送到移动端，移动端收到事件后刷新消息列表、通讯录或待处理好友申请状态。

### 5.10 NFC 读取事件记录

后端保存 NFC 读取事件，包括读取内容、解析结果、用户关联等信息，用于后续统计、测试复盘和异常排查。

## 6. 典型使用流程

1. 用户打开次聊 App。
2. 用户通过用户名密码或手机号 mock 验证码注册账号。
3. 用户登录后进入主界面。
4. 用户设置昵称、头像和个性签名。
5. 用户使用 NFC 触碰好友邀请标签，或手动输入 invite 内容。
6. 软件解析目标用户并展示好友申请入口。
7. 用户发送好友申请。
8. 对方登录后在待处理申请中同意。
9. 系统为双方建立双向好友关系。
10. 双方进入通讯录选择对方，开始一对一文字聊天。
11. 后端保存聊天记录并通过 WebSocket 推送新消息。

## 7. 数据设计概要

| 数据对象 | 说明 |
| --- | --- |
| 用户 | 保存用户 ID、用户名、手机号、昵称、头像、个性签名等资料 |
| 凭证 | 保存账号登录凭证和密码校验相关信息 |
| 会话 | 保存登录会话和访问令牌 |
| 联系人 | 保存双向好友关系 |
| 好友申请 | 保存申请人、接收人、状态和处理时间 |
| 会话记录 | 保存一对一聊天会话 |
| 消息 | 保存消息发送人、接收人、内容和时间 |
| NFC 事件 | 保存 NFC 读取内容、解析结果和关联用户 |

## 8. 接口概要

| 接口类别 | 示例路径 | 说明 |
| --- | --- | --- |
| 健康检查 | `GET /health` | 检查后端服务是否可用 |
| 注册登录 | `/auth/register/username`、`/auth/login` | 账号注册、登录和会话恢复 |
| 用户资料 | `/auth/me`、头像相关接口 | 获取和更新当前用户信息 |
| 联系人 | `/contacts` | 获取通讯录或添加联系人 |
| 好友搜索 | `/friend-search` | 按账号信息搜索目标用户 |
| 好友申请 | `/friend-requests` | 创建、同意、拒绝好友申请 |
| NFC 事件 | `/nfc/read-events` | 记录 NFC 读取事件 |
| 消息 | `/messages/{contactId}`、`/messages` | 读取和发送聊天消息 |
| 实时通信 | WebSocket | 推送消息和联系人状态变化 |

## 9. 安全与权限说明

软件使用登录令牌识别当前用户。需要登录的接口通过 Bearer token 访问。移动端请求 NFC 权限用于读取 NDEF 标签。当前版本为原型验证版本，未包含生产级风控、审计、真实短信服务、第三方登录和后台管理系统。

## 10. 版本说明

V1.0 版本完成以下核心能力：

- 用户名密码注册和登录。
- 手机号 mock 短信注册。
- 自动登录、退出登录、修改密码。
- 个人资料、头像和个性签名维护。
- 用户搜索、好友申请、同意和拒绝。
- NFC invite 读取和好友申请流程。
- NFC 读取事件记录。
- 通讯录和双向好友关系。
- 一对一文字聊天。
- WebSocket 实时消息推送。
- 后端数据库迁移和数据持久化。

## 11. 未包含功能

当前版本不包含真实短信服务、密码找回、第三方登录、群聊、图片消息、语音消息、后台管理系统、生产级安全审计、生产级数据库备份和应用商店正式发布流程。

## 12. 结论

次聊 NFC 社交聊天软件 V1.0 已形成较完整的移动端与后端组合，实现了从账号注册、NFC 邀请、好友申请、通讯录建立到一对一实时聊天的主链路，具备作为软件著作权登记材料的功能说明和源程序整理基础。
